# Extend and Contribute to Sunbird

#### Priority definitions

All issues filed in the Sunbird issue tracker are assigned a priority. The priority determines the expected timeline for a fix.

| Priority                       | Definition                                                                                      | Examples                                                                         |
| ------------------------------ | ----------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| P1 — Fix immediately           | Security breaches, version incompatibilities, build breaks, regressions from a previous release | Log4Shell-type vulnerability; certificates that stopped working after an upgrade |
| P2 — Fix in next release       | Breakage of a major feature used by most adopters; issue blocking a strategic initiative        | Course enrolment failing for all users; a feature required for a national launch |
| P3 — Fix when bandwidth allows | All other issues and enhancements that align with the roadmap                                   | Minor UI inconsistencies; nice-to-have feature additions                         |
