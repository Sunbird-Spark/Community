# Is Spark right for you?

| Your scenario                                                                            | Recommendation                                                                                               |
| ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| You need a fully functional LMS for up to 100k users with low infrastructure cost        | Spark base install — ideal fit                                                                               |
| You need deep custom analytics, cohort comparisons, and time-series reports from day one | Deploy Spark + enable the Obsrv add-on                                                                       |
| You need QR code scanning to link physical materials to digital content                  | Deploy Spark + enable the DIAL add-on                                                                        |
| You need discussion forums for learner-to-learner interaction                            | Deploy Spark + enable the Discussion Forum add-on                                                            |
| You need to scale to 1 million+ users                                                    | Spark's architecture supports this — plan your Kubernetes cluster for horizontal scaling                     |
| You need a fully managed SaaS platform with no self-hosting                              | Spark is self-hosted. Consider working with a managed hosting partner or Sunbird community implementers      |
| You need to white-label the platform with your own branding                              | Spark supports multi-theme configuration — your logo, colours, and domain                                    |
| You are migrating from Sunbird ED                                                        | Spark is the direct successor. See the [ED → Spark Migration Guide](../use/ed-greater-than-spark-migration/) |
| You want to embed Spark's content players or editors into your own application           | The V2 web components are embeddable in any web application — independently of the full Spark deployment     |
