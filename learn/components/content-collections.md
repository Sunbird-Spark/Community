# Content collections

Content in Spark can be organised into collections that give learners a structured, progressive learning experience.

### Collection types at a glance

| | **Course** | **Playlist** | **Digital textbook** |
|---|---|---|---|
| Account required | Yes | No | No |
| Progress tracked | Yes | No | No |
| Certificate on completion | Yes | No | No |
| Batch management | Yes | No | No |
| Typical use case | Structured training programmes | Resource libraries, reference material | Curriculum-aligned content for browse-and-learn |

### Trackable collections — Courses

Courses require a learner account and track individual progress.

**They support:**

* **Batch management** — a course runs as one or more batches, each with its own enrolment window, start date, and end date. Open batches allow self-enrolment at any time; closed batches require an administrator to add learners.
* Progress tracking at both lesson and course level
* Certificate issuance when a learner completes `100%` of the course
* Assessment integration with QuML-based assessments embedded within a course

### Non-trackable collections

Playlists and digital textbooks are open access. They do not require a learner account and do not track progress.

They are suitable for reference material, resource libraries, and browse-and-learn use cases.
