# User accounts and access

#### Accounts and authentication

* Self-registration with email or phone, with OTP-based verification
* Single Sign-On with Google on both the web portal and mobile app
* Configurable Terms and Conditions during registration
* RBAC with roles such as Learner, Content Creator, Content Reviewer, Book Creator, Book Reviewer, Programme Manager, Organisation Administrator, and Report Administrator

#### Learning passbook and user report

Every logged-in learner has access to a personal **User Report** page showing:

* A summary of courses enrolled, in progress, completed, and certificates earned
* A course-by-course progress table with enrolled date and last accessed date
* A certificates section with certificate IDs and issue dates
* An assessment history section
* CSV export for all sections

#### Notifications

* In-app notifications on the web portal
* Push notifications on the mobile app
* Notification triggers for batch start reminders, course completion, certificate issuance, and programme announcements

#### Discussion forums

{% hint style="info" %}
**Add-on required:** Discussion forums are an optional Spark add-on. They are disabled by default. The forum is powered by NodeBB `4.x`.
{% endhint %}

When enabled, discussion forums let learners engage with peers and instructors around course content. Forums can be scoped to a specific course or opened to the full platform community.

#### Groups

{% hint style="info" %}
**Add-on required:** Groups functionality is an optional Spark add-on. It is disabled by default.
{% endhint %}

Groups let administrators create cohorts of learners and assign specific content or courses to them. This supports targeted programme delivery, such as assigning a training module to all newly onboarded employees in one department.
