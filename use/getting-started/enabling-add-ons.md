# Enabling add-ons

Add-ons extend Sunbird's core functionality. They are optional, independently deployable, and can be installed at any time after the base installation is complete.

***

### How It Works

Base layer `install.sh` is installed, it provisions your desired cloud infrastructure and deploys core services.

Once the base install is complete, add-ons can be deployed independently using their own `addon.sh` script. Each add-on's script picks up the values already generated by the base install and deploys the add-on's Helm charts onto the running cluster. Add-ons can be installed in two ways, matching the same two methods available in the one-click installer.

#### Method 1 вЂ” GitHub Actions

This is the recommended approach if you set up Sunbird using the GitHub Actions method. Your private config repo already contains the `sunbird-spark-addons.yaml` workflow for deploying add-ons.

### Exception вЂ” DIAL Add-on

DIAL requires additional steps compared to other add-ons. Unlike the rest, it provisions cloud infrastructure (a storage container) via OpenTofu as part of its setup, and requires a core service to be redeployed afterward to pick up the new configuration.

**Step 1 вЂ” Run the DIAL addon script**

Navigate to the DIAL addon directory and run the install script. This provisions the required cloud container via OpenTofu.

{% hint style="info" %}
DIAL requires a infrastructure deployment, uncomment **`provision_resources`** in the **`./script/addon.sh`** file for OpenTofu or infrastructure provisioning.&#x20;

```shellscript
install() {
    # provision_resources
    deploy_chart
}
```
{% endhint %}

```bash
cd addons/dial
./script/addon.sh install azure
```

**Step 2 вЂ” Enable the DIAL flag in `global-values.yaml` in** [**spark-ed-installer**](https://github.com/Sunbird-Spark/sunbird-spark-installer/blob/develop/opentofu/azure/template/global-values.yaml) **repo**

Once the addon is deployed, set the dial dependency flag to inform core services that DIAL is active:

```yaml
deployed_dial_addon:  &deployed_dial_addon  "true"
```

**Step 3 вЂ” Edit `install.sh` to target only `knowledgebb`**

Open `install.sh` and update the `install_helm_components` function to redeploy only `knowledgebb`:

```bash
function install_helm_components() {
    install_component knowledgebb
}
```

> After deployment revert the `install_helm_components` function in `install.sh` back to its original state after this step to avoid accidentally redeploying only `knowledgebb` in future runs

**Step 4 вЂ” Rerun the installer for `knowledgebb`**

```bash
./install.sh install_helm_components
```

This redeploys `knowledgebb` with the updated DIAL configuration included.
